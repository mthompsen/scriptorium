# Aurelia Corp Information Security Policy

## Passwords and authentication

All passwords must be at least 14 characters long and generated by the
approved password manager. Multi-factor authentication is mandatory on every
internal system; hardware security keys are issued to all staff and required
for production access. Shared accounts are prohibited.

## Device security

Company laptops use full-disk encryption enforced by MDM. Personal devices
may not access production systems. Lost or stolen devices must be reported
immediately so remote wipe can be triggered.

## Incident reporting

Suspected security incidents must be reported to secops@aurelia.example
within 24 hours of discovery. The security operations team triages within
one business day and classifies incidents as SEV-1 through SEV-4. Employees
must not attempt to investigate or remediate incidents themselves.

## Phishing

Report suspicious emails with the Report Phish button. The security team
runs quarterly phishing simulations; repeated failures trigger mandatory
refresher training.

## Data classification

Data is classified as Public, Internal, Confidential, or Restricted.
Restricted data (customer PII, credentials, financial records) may only be
stored in approved systems and never in personal cloud storage.
